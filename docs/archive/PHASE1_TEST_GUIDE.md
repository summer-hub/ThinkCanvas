## 📋 测试前准备

### 1. 确保开发服务器运行

```bash
npm run dev
```

访问: http://localhost:5174/

## 🧪 测试步骤

### 测试 1: 基础上传功能

**步骤**:
1. 点击 Header 的"上传"按钮
2. 选择 `test-markdown.md` 文件
3. **不勾选** "启用 AI 智能分析"
4. 观察上传过程

**预期结果**:
- ✅ 显示上传进度
- ✅ 创建一个简单的文本节点
- ✅ 节点内容包含文件名和部分内容
- ✅ 上传完成后自动关闭弹窗

---

### 测试 2: AI 智能分析（Markdown）

**步骤**:
1. 点击 Header 的"上传"按钮
2. 选择 `test-markdown.md` 文件
3. **勾选** "启用 AI 智能分析"
4. 观察上传过程

**预期结果**:

**阶段 1: 上传中** 📤
- ✅ 显示"正在上传文件..."
- ✅ 图标变为 📤

**阶段 2: 分析中** 🔍
- ✅ 显示"AI 正在分析文档结构..."
- ✅ 图标变为 🔍
- ⏱️ 预计耗时: 5-15 秒

**阶段 3: 生成图谱** 🎨
- ✅ 显示"正在生成知识图谱..."
- ✅ 图标变为 🎨
- ⏱️ 预计耗时: 1-2 秒

**阶段 4: 索引中** 🔄
- ✅ 显示"正在索引文件..."
- ✅ 显示进度条
- ✅ 显示百分比
- ⏱️ 预计耗时: 3-10 秒

**阶段 5: 完成** ✅
- ✅ 显示"上传完成！"
- ✅ 图标变为 ✅
- ✅ 自动关闭弹窗

**生成的节点**:

1. **中心节点（根节点）**
   - 位置: 画布中心
   - 颜色: 紫色
   - 图标: 🎯
   - 内容: 文档标题 + 摘要
   - 尺寸: 较大 (350x200)

2. **章节节点（3-4 个）**
   - 位置: 环绕中心节点
   - 颜色: 蓝色
   - 图标: 📑
   - 内容: 章节标题 + 摘要 + 占比
   - 尺寸: 中等 (280x180)
   - 连接: 连接到中心节点

3. **要点节点（若干个）**
   - 位置: 围绕章节节点
   - 颜色: 绿色
   - 图标: 📝
   - 内容: 具体要点
   - 尺寸: 较小 (200x100)
   - 连接: 连接到对应章节节点

4. **洞察节点（3 个）**
   - 位置: 中心节点上方
   - 颜色: 橙色
   - 图标: 💡
   - 内容: AI 生成的深层洞察
   - 尺寸: 中等 (300x150)
   - 连接: 连接到中心节点

**布局验证**:
- ✅ 节点不重叠
- ✅ 章节节点均匀分布在圆周上
- ✅ 连接线清晰可见
- ✅ 整体布局美观

## 🐛 已知问题

### 问题 1: AI 返回格式不正确

**现象**: AI 返回的不是有效的 JSON

**原因**: AI 可能在 JSON 前后添加额外文字

**解决**: 代码中已添加正则提取 JSON

**验证**: 查看控制台是否有 JSON 解析错误

### 问题 2: 章节识别不准确

**现象**: 纯文本文档无法识别章节

**原因**: 快速分析只支持 Markdown 标题

**解决**: 这是预期行为，需要 AI 分析才能识别

**建议**: 使用 Markdown 格式的测试文件

### 问题 3: 节点过多

**现象**: 生成了太多节点（>50 个）

**原因**: 文档章节和要点过多

**解决**: 可以限制要点节点的生成

**优化**: Phase 2 中添加节点数量控制
