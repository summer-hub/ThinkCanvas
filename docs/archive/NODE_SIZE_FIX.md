# 🔧 节点大小问题修复

## 问题描述

AI 响应节点内容过多时，文字会超出节点边界，因为节点使用固定的小尺寸（200x100）。

## 解决方案

### 1. 添加滚动条支持

**文件**：`src/components/PonderNode.tsx`

**修改**：
- 内容区域添加 `overflow-y-auto` 类
- 添加 `custom-scrollbar` 类使用自定义滚动条样式
- 添加 `overflow-hidden` 到父容器防止溢出
- 设置 `maxHeight: 600px` 限制最大高度

**效果**：
- 内容超出时显示滚动条
- 可以滚动查看完整内容
- 节点不会无限增长

---

### 2. AI 节点使用更大默认尺寸

**文件**：`src/store/canvasStore.ts`

**修改**：
```typescript
// 之前：所有节点都是 200x100
width: 200,
height: 100,

// 现在：AI 节点更大
const defaultWidth = type === 'ai-response' ? 300 : 200;
const defaultHeight = type === 'ai-response' ? 200 : 100;
```

**效果**：
- AI 响应节点默认 300x200（更大）
- 普通文本节点保持 200x100
- 可以容纳更多内容

---

### 3. 支持手动调整大小

**已有功能**：
- 选中节点后可以拖拽角落调整大小
- 最小尺寸：160x60
- 最大高度：600px

---

## 使用方法

### 自动调整（推荐）

1. **创建 AI 响应节点**
   - 选择一个节点
   - 在 AI 面板输入问题
   - AI 响应会自动创建更大的节点（300x200）

2. **查看长内容**
   - 如果内容超出节点，会显示滚动条
   - 鼠标悬停在节点上，滚动查看完整内容

### 手动调整

1. **选中节点**
   - 点击节点使其被选中
   - 节点四角会出现调整手柄

2. **拖拽调整**
   - 拖拽右下角的手柄
   - 向下拖动增加高度
   - 向右拖动增加宽度

3. **尺寸限制**
   - 最小：160x60
   - 最大高度：600px
   - 宽度无限制

---

## 节点尺寸建议

### 普通文本节点
- **默认**：200x100
- **短文本**：保持默认
- **中等文本**：调整到 200x150
- **长文本**：调整到 250x200

### AI 响应节点
- **默认**：300x200（自动）
- **短回复**：保持默认
- **中等回复**：调整到 300x250
- **长回复**：调整到 350x300 或更大

### 文件内容节点
- **默认**：200x100
- **建议**：调整到 300x250
- **长文档**：调整到 400x400

## 常见问题

### Q1: 为什么 AI 响应节点还是显示不全？

**原因**：内容太长，超过默认高度

**解决**：
1. 选中节点
2. 拖拽右下角向下调整高度
3. 或者滚动查看完整内容

---

### Q2: 如何让所有节点都使用更大的默认尺寸？

**方法 1**：修改代码
```typescript
// src/store/canvasStore.ts
const defaultWidth = 300;  // 改为 300
const defaultHeight = 200; // 改为 200
```

**方法 2**：创建后手动调整
- 选中节点
- 拖拽调整到想要的大小

---

### Q3: 滚动条不显示？

**原因**：内容未超出节点高度

**检查**：
1. 节点高度是否足够大
2. 内容是否真的很长
3. 浏览器是否支持自定义滚动条

---

### Q4: 如何隐藏滚动条？

**修改**：`src/components/PonderNode.tsx`
```typescript
// 移除 overflow-y-auto 类
className="flex-1 text-sm text-canvas-text whitespace-pre-wrap break-words"
```

**注意**：这样内容会被截断
