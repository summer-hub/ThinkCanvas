# ↩️ 撤销/重做功能文档

## 功能概述

Ponder AI 现在支持完整的撤销/重做功能，让你可以自由地尝试想法而不用担心犯错。

---

## 🎯 功能特性

### 支持的操作
- ✅ 创建节点
- ✅ 删除节点
- ✅ 编辑节点内容
- ✅ 改变节点颜色
- ✅ 创建连接
- ✅ 删除连接

### 不支持的操作
- ❌ 节点位置移动（实时操作，不记录历史）
- ❌ 节点大小调整（实时操作，不记录历史）
- ❌ AI 对话（不可撤销）
- ❌ 选择操作（不需要撤销）

---

## ⌨️ 快捷键

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+Z` (Win/Linux) | 撤销 | 撤销上一步操作 |
| `Cmd+Z` (Mac) | 撤销 | 撤销上一步操作 |
| `Ctrl+Shift+Z` (Win/Linux) | 重做 | 重做已撤销的操作 |
| `Cmd+Shift+Z` (Mac) | 重做 | 重做已撤销的操作 |
| `Ctrl+Y` (Win/Linux) | 重做 | 重做的替代快捷键 |
| `Cmd+Y` (Mac) | 重做 | 重做的替代快捷键 |

---

## 🖱️ UI 按钮

### Header 中的按钮
- **↶ 撤销按钮**：点击撤销上一步操作
- **↷ 重做按钮**：点击重做已撤销的操作
- 按钮状态：
  - 可用：正常颜色
  - 不可用：灰色半透明（无历史记录时）

## 📚 使用示例

### 示例 1：撤销删除
```
1. 创建一个节点："重要想法"
2. 不小心按 Delete 删除了
3. 按 Ctrl+Z 撤销
4. ✅ 节点恢复了！
```

### 示例 2：尝试不同颜色
```
1. 创建一个节点
2. 改成红色 → 不喜欢
3. 按 Ctrl+Z 撤销
4. 改成蓝色 → 还是不喜欢
5. 按 Ctrl+Z 撤销
6. 改成紫色 → 完美！
```

### 示例 3：重做操作
```
1. 创建节点 A
2. 创建节点 B
3. 按 Ctrl+Z 撤销（B 消失）
4. 按 Ctrl+Z 再撤销（A 也消失）
5. 按 Ctrl+Shift+Z 重做（A 回来了）
6. 按 Ctrl+Shift+Z 再重做（B 也回来了）
```

### 示例 4：新操作清空重做栈
```
1. 创建节点 A
2. 创建节点 B
3. 按 Ctrl+Z 撤销（B 消失）
4. 创建节点 C（新操作）
5. ❌ 无法重做 B（重做历史被清空）
```

## 🏗️ 技术实现

### 架构设计

#### 1. 历史栈结构
```typescript
interface CanvasSnapshot {
  nodes: PonderNode[];
  edges: PonderEdge[];
  timestamp: number;
}

interface HistoryStore {
  past: CanvasSnapshot[];    // 撤销栈
  future: CanvasSnapshot[];  // 重做栈
  maxHistorySize: 50;        // 最大历史记录数
}
```

#### 2. 工作流程

**撤销流程**：
```
1. 从 past 栈取出最后一个快照
2. 将当前状态推入 future 栈
3. 恢复快照到画布
```

**重做流程**：
```
1. 从 future 栈取出第一个快照
2. 将当前状态推入 past 栈
3. 恢复快照到画布
```

**新操作流程**：
```
1. 保存当前状态到 past 栈
2. 清空 future 栈
3. 执行新操作
```

### 核心文件

#### `src/store/historyStore.ts`
历史记录管理 Store
- `pushHistory()` - 保存快照
- `undo()` - 撤销操作
- `redo()` - 重做操作
- `canUndo()` - 是否可撤销
- `canRedo()` - 是否可重做

#### `src/store/canvasStore.ts`
画布 Store（集成历史记录）
- `saveSnapshot()` - 保存当前状态
- `restoreSnapshot()` - 恢复快照
- 所有修改操作都会自动保存快照

#### `src/hooks/useKeyboardShortcuts.ts`
快捷键处理
- 监听 Ctrl/Cmd+Z
- 监听 Ctrl/Cmd+Shift+Z
- 监听 Ctrl/Cmd+Y

#### `src/components/Header.tsx`
UI 按钮
- 撤销按钮
- 重做按钮
- 按钮状态管理

## ⚙️ 配置选项

### 历史记录限制
```typescript
maxHistorySize: 50  // 最多保存 50 步历史
```

**为什么限制？**
- 防止内存占用过大
- 50 步对大多数用户足够
- 超过限制会自动删除最旧的记录

### 修改限制
编辑 `src/store/historyStore.ts`：
```typescript
maxHistorySize: 100  // 改为 100 步
```

## 🐛 已知限制

### 1. 不记录位置变化
**原因**：拖拽节点会产生大量中间状态，记录所有位置会导致：
- 历史栈快速填满
- 内存占用激增
- 撤销体验不佳（一次只移动一点点）

**解决方案**：只记录创建、删除、编辑等离散操作

### 2. 不记录尺寸调整
**原因**：同位置变化，调整大小是连续操作

**解决方案**：只在调整完成时保存最终尺寸

### 3. AI 对话不可撤销
**原因**：
- AI 响应需要 API 调用，无法真正"撤销"
- 撤销 AI 节点会导致对话历史混乱

**解决方案**：可以手动删除 AI 节点（可撤销）

## 🎯 最佳实践

### 1. 大胆尝试
- 不用担心犯错，随时可以撤销
- 尝试不同的布局和颜色
- 快速迭代想法

### 2. 善用快捷键
- 记住 Ctrl+Z 和 Ctrl+Shift+Z
- 比点击按钮快得多
- 提高工作效率

### 3. 注意历史限制
- 超过 50 步后，最旧的记录会被删除
- 重要操作后可以手动保存（导出功能开发中）

### 4. 理解重做栈清空
- 新操作会清空重做历史
- 这是标准的撤销/重做行为
- 类似于 Photoshop、Word 等软件

## 📈 性能考虑

### 内存占用
- 每个快照约 1-10KB（取决于节点数量）
- 50 个快照约 50-500KB
- 对现代浏览器来说可以忽略不计

### 性能优化
- ✅ 使用深拷贝避免引用问题
- ✅ 限制历史记录数量
- ✅ 防抖保存避免频繁操作
- ✅ 只在必要时保存快照
