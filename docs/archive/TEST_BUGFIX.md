# 🧪 Bug 修复测试指南

## 测试目标
验证节点无限变大的 Bug 已被修复。

---

## 测试步骤

### 1. 刷新页面
```
在浏览器中按 F5 或 Cmd+R 刷新页面
确保加载最新的代码
```

### 2. 创建节点测试

#### 测试 A: 基础创建
1. 双击画布空白处
2. 输入文字："测试节点"
3. 按 Enter 确认
4. **观察 5 秒钟**

**预期结果**: ✅ 节点保持稳定，不会变大

**失败标志**: ❌ 节点不断变大

---

#### 测试 B: 多个节点
1. 创建 3-5 个节点
2. 不要选中任何节点
3. **观察 10 秒钟**

**预期结果**: ✅ 所有节点保持稳定

**失败标志**: ❌ 任何节点变大

---

#### 测试 C: 选中节点
1. 创建一个节点
2. 点击选中该节点
3. **观察 5 秒钟**

**预期结果**: ✅ 节点保持稳定，显示调整手柄

**失败标志**: ❌ 节点变大

---

### 3. 手动调整尺寸测试

#### 测试 D: 调整大小
1. 创建一个节点
2. 选中节点
3. 拖动右下角的调整手柄
4. 放大节点到约 300x150
5. 释放鼠标
6. **观察 5 秒钟**

**预期结果**: 
- ✅ 调整过程流畅
- ✅ 释放后尺寸固定
- ✅ 不会继续变大

**失败标志**: 
- ❌ 释放后继续变大
- ❌ 调整过程卡顿

---

#### 测试 E: 持久化
1. 创建并调整一个节点的大小
2. 刷新页面（F5）
3. 检查节点尺寸

**预期结果**: ✅ 节点尺寸保持不变

**失败标志**: ❌ 节点恢复默认大小或变大

---

### 4. AI 对话测试

#### 测试 F: AI 响应节点
1. 创建一个节点："什么是人工智能？"
2. 选中节点，打开 AI 面板
3. 发送消息给 AI
4. 等待 AI 创建响应节点
5. **观察新节点 5 秒钟**

**预期结果**: 
- ✅ AI 节点正常创建
- ✅ AI 节点保持稳定
- ✅ 显示 🤖 图标

**失败标志**: 
- ❌ AI 节点不断变大
- ❌ 节点创建失败

---

### 5. 性能测试

#### 测试 G: CPU 占用
1. 打开浏览器任务管理器
   - Chrome: Shift+Esc
   - Firefox: about:performance
2. 创建 5 个节点
3. 观察 CPU 占用率

**预期结果**: 
- ✅ CPU 占用 < 10%（空闲时）
- ✅ 无持续高占用

**失败标志**: 
- ❌ CPU 持续 > 50%
- ❌ 风扇狂转

---

#### 测试 H: 内存占用
1. 创建 10 个节点
2. 观察内存占用
3. 删除所有节点
4. 观察内存是否释放

**预期结果**: 
- ✅ 内存占用合理（< 200MB）
- ✅ 删除后内存下降

**失败标志**: 
- ❌ 内存持续增长
- ❌ 删除后不释放

## 如果测试失败

### 1. 确认代码已更新
```bash
# 检查最新提交
git log -1

# 确认修改的文件
git diff HEAD~1
```

### 2. 清除缓存
```bash
# 清除浏览器缓存
Ctrl+Shift+Delete (Windows/Linux)
Cmd+Shift+Delete (Mac)

# 或者硬刷新
Ctrl+F5 (Windows/Linux)
Cmd+Shift+R (Mac)
```

### 3. 重启开发服务器
```bash
# 停止服务器
Ctrl+C

# 重新启动
npm run dev
```

### 4. 检查控制台错误
```
打开浏览器开发者工具 (F12)
查看 Console 标签
截图任何错误信息
```

### 5. 报告问题
如果问题仍然存在，请提供：
- 测试结果截图
- 控制台错误信息
- 浏览器和版本
- 操作系统
- 复现步骤
