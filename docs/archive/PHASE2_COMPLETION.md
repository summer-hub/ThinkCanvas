## 🐛 已知限制

### 1. 分析依赖节点颜色

**现象**: 分析面板通过节点颜色识别类型

**原因**: 
- 根节点: 紫色 (#9333ea)
- 章节节点: 蓝色 (#3b82f6)
- 洞察节点: 橙色 (#f59e0b)

**影响**: 
- 手动创建的节点可能无法识别
- 需要使用 AI 分析生成的节点

**解决**: 
- Phase 1 生成的节点自动设置颜色
- 手动节点可以通过右键菜单设置颜色

### 2. 文件列表不自动刷新

**现象**: 上传新文件后需要手动刷新

**原因**: 资源面板不监听文件变化

**解决**: 点击"刷新列表"按钮

**优化**: 可以添加文件变化监听

### 3. 分析面板不自动更新

**现象**: 修改节点后需要手动刷新分析

**原因**: 分析只在节点数组变化时触发

**解决**: 点击"刷新分析"按钮

**优化**: 可以添加节点内容变化监听

## 📝 开发笔记

### 设计决策

1. **为什么使用固定宽度？**
   - 保证内容可读性
   - 避免布局抖动
   - 折叠后节省空间

2. **为什么分析依赖颜色？**
   - 简单高效
   - 与 Phase 1 生成的节点一致
   - 易于扩展

3. **为什么需要手动刷新？**
   - 避免频繁计算
   - 用户控制更新时机
   - 性能考虑

### 技术挑战

1. **布局响应式**
   - 使用 flex 布局
   - 固定宽度 + flex-1
   - 折叠状态切换

2. **状态管理**
   - Zustand 全局状态
   - localStorage 持久化
   - 组件间通信

3. **性能优化**
   - 按需分析
   - 手动刷新
   - 避免不必要的重渲染
