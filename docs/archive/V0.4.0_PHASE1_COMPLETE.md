// 自动在句子边界分块，保持语义完整

## 🔧 使用示例

### 索引文件
```typescript
import { indexFile } from '@/lib/rag';

// 上传文件后自动索引
await indexFile(uploadedFile, apiKey, (current, total) => {
  console.log(`Indexing: ${current}/${total} chunks`);
});
```

### 语义搜索
```typescript
import { searchSimilarChunks } from '@/lib/rag';

// 搜索相关内容
const results = await searchSimilarChunks(
  "什么是机器学习？",
  apiKey,
  topK: 3
);

results.forEach(chunk => {
  console.log(`${chunk.fileName}: ${chunk.content}`);
  console.log(`Similarity: ${chunk.similarity}`);
});
```

### 检索上下文
```typescript
import { retrieveContext, enhancePrompt } from '@/lib/rag';

// 检索相关上下文
const { context, sources } = await retrieveContext(
  "请解释 @paper.pdf 中的核心算法",
  apiKey
);

// 增强 AI Prompt
const enhancedPrompt = enhancePrompt(userMessage, context);
```

## 🎮 工作流程

### 文件上传 → 索引流程
```
1. 用户上传文件
   ↓
2. 解析文件内容
   ↓
3. 文本分块（~500 字符/块）
   ↓
4. 批量生成嵌入（OpenAI API）
   ↓
5. 存储到 IndexedDB
   ↓
6. 创建向量索引
   ↓
7. 完成！文件可被搜索
```

### AI 对话 → RAG 流程（下一阶段）
```
1. 用户发送消息（可能包含 @文件引用）
   ↓
2. 解析 @文件引用
   ↓
3. 生成查询嵌入
   ↓
4. 语义搜索相关 chunks
   ↓
5. 格式化上下文
   ↓
6. 增强 AI Prompt
   ↓
7. 调用 AI API
   ↓
8. 返回带引用的回复
```

## 💡 技术决策

### 为什么选择纯前端方案？

1. **零配置**
   - 用户无需安装 Python
   - 无需配置后端服务
   - 开箱即用

2. **隐私保护**
   - 所有数据存储在本地
   - 仅嵌入生成需要 API 调用
   - 用户完全控制数据

3. **开发效率**
   - 与现有架构一致
   - 无需学习新技术栈
   - 快速迭代

4. **成本低**
   - OpenAI Embeddings 极便宜
   - 10 个文档 < $0.01
   - 用户可接受

### 为什么选择 text-embedding-3-small？

1. **性价比高**
   - $0.00002 / 1K tokens
   - 比 large 模型便宜 6.5 倍

2. **性能足够**
   - 1536 维向量
   - 准确度满足需求

3. **速度快**
   - API 响应快
   - 向量计算快
