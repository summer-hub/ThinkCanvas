## 📝 开发笔记

### 当前架构优势
1. **纯前端实现**：无需后端服务器，降低部署复杂度
2. **本地优先**：数据存储在用户本地，隐私安全
3. **模块化设计**：组件职责清晰，易于扩展
4. **类型安全**：TypeScript 提供良好的开发体验

### 需要考虑的技术债务
1. **缺少后端**：文件解析和 RAG 功能可能需要后端支持
2. **状态管理**：随着功能增加，可能需要更复杂的状态管理
3. **性能优化**：大量节点时的渲染性能
4. **测试覆盖**：目前没有单元测试和集成测试

### 技术决策点
- **是否添加后端**：如果要实现 RAG 和文件解析，可能需要
  - 选项 1：保持纯前端，使用浏览器 API 和云服务
  - 选项 2：添加 Python FastAPI 后端，本地部署
  - 选项 3：混合方案，核心功能前端，高级功能后端

- **向量数据库选择**：
  - Chroma：轻量，易于集成
  - PGVector：需要 PostgreSQL，更强大
  - Pinecone：云服务，无需部署
